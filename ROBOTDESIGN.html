Robot Design Overview

How we built a robot that works the same way every time

We explain our build, strategy, and programming in simple steps so anyone can understand our process.

Key takeaway: We focused on repeatable results, not just one good run.

What we built

A stable base, reliable attachments, and clean wiring for quick fixes.

How it works

Sensors guide turns and distance so the robot stays accurate.

How we improved

Build Evolution

From an uneven base to a stable robot

We simplified the motor layout and attachments so the robot is easier to build, check, and repair.

Key takeaway: A simple, stable base made every run more reliable.
Strategy

We grouped missions to save time and reduce risk

Instead of many long runs, we combined nearby missions into short, safe runs.

Key takeaway: Fewer, smarter runs = fewer errors.

Programming (Pybricks)

Simple code loops keep the robot on track

We use fast code loops so the robot can correct small errors quickly.

Key takeaway: Faster loops = faster corrections.
Short code example
# simple loop (example)
while True:
  error = target - gyro.angle()
  power = kp * error
  drive.dc(power, power)
  wait(10)
